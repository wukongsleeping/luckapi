## 报告漏洞

我们非常重视 LuckApi 的安全。发现安全漏洞时，请通过以下 **私密渠道** 报告：

- **邮件**：`security@luckapi.dev`
- **主题格式**：`[SECURITY] <简短描述>`
- **请勿**在公开 Issue / Discussion /社交媒体 中披露漏洞细节

### 报告内容建议

1. 漏洞类型（如：认证绕过、注入、越权访问等）
2. 复现步骤与 PoC 代码
3. 漏洞影响范围及潜在危害
4. 运行环境（Docker / 原生部署、Python 版本等）

### 处理流程

| 阶段 | 时间 | 说明 |
|------|------|------|
| 确认收到的回复 | 48 小时内 | 确认收到报告，指定安全联系人 |
| 漏洞评估 | 3 个工作日 | 评估严重程度和优先级 |
| 补丁开发 | 7 天内（Critical） / 14 天内（High） / 30 天内（Medium/Low） | 开发并测试修复方案 |
| 发布补丁 | 评估完成后 | 通过邮件通知报告者，并在 Changelog 中披露 |

### 回复时间

- **Critical / High**: 48 小时内首次响应
- **Medium**: 3 个工作日内首次响应
- **Low**: 1 周内首次响应

## 安全最佳实践

部署 LuckApi 时请遵循：

1. **不在公开仓库中提交 `.env` 文件**（已加入 `.gitignore`）
2. **使用 HTTPS 访问前端和管理 API**（Nginx 配置 SSL 证书）
3. **定期更新依赖**：`pip install -r requirements.txt --upgrade`
4. **修改默认管理员密码**
5. **使用强 FERNET_KEY**（32 字节 URL-safe base64 编码）
6. **限制数据库和 Redis 的访问端口**（生产环境不建议暴露在公网）
