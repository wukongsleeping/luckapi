## Phase 1: 紧急安全修复

### P1-1: 密码哈希升级为 bcrypt [CRITICAL]
- **文件**: `backend/app/core/security.py`
- **当前**: SHA-256 + 静态 salt（`luckapi-salt`）
- **目标**: passlib with bcrypt（已在 requirements.txt 中）
- **影响**: 所有用户密码需要重新哈希（升级登录时自动升级）
- **状态**: ✅ 已完成

### P1-2: Redis URL 端口修复 [CRITICAL]
- **文件**: `backend/app/core/config.py:41`
- **当前**: `redis://:pass@host:0`（缺少端口号）
- **目标**: `redis://:pass@host:6379/0`
- **状态**: ✅ 已完成

### P1-3: API Key 不在响应中泄露 [HIGH]
- **文件**: `backend/app/schemas/model.py`, `backend/app/schemas/admin_model.py`
- **当前**: `UserModelOut.api_key: str` 明文返回
- **目标**: 移除 `api_key` 字段或用掩码（`sk-xxxx...abcd`）
- **状态**: ✅ 已完成

### P1-4: 硬编码默认凭证移除 [HIGH]
- **文件**: `backend/app/core/config.py`, `backend/db_init.py`
- **当前**: `SECRET_KEY="luckapi-secret-key-..."`, `MYSQL_PASSWORD="root123"`, `DEFAULT_PASSWORD="admin123"`
- **目标**: 强制使用环境变量，无安全默认值
- **详情**:
  - `config.py` 的 `SECRET_KEY` / `MYSQL_PASSWORD` 改为无默认值（空字符串）
  - `db_init.py` 在密码未配置环境变量时拒绝创建默认用户，或提示用户设置
- **状态**: ✅ 已完成

### P1-5: API Key 复制截断修复 [移动端-HIGH]
- **文件**: `uniapp/pages/users/userDetail.vue`
- **当前**: `copyKey()` 复制的是被截断的 `this.apiKey`（前 20 字符 + "..."）
- **目标**: 新增"显示完整 key"的切换功能，复制使用完整 key
- **状态**: ✅ 已完成
